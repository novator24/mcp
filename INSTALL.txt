# Переустановка WSL2 с Ubuntu и установка Cursor на Windows

Это пошаговая инструкция с нуля: удалить старую WSL-конфигурацию, поставить WSL2 заново, установить Ubuntu и настроить Cursor для работы с проектами внутри WSL.

## Что понадобится

- Windows 10 или Windows 11 с установленными последними обновлениями. [page:2]
- Права администратора в Windows. [page:2]
- Подключение к интернету.
- Свободное место на диске.

## 1. Удалите старую WSL и Ubuntu

Если вы хотите начать совсем с нуля, сначала удалите старый дистрибутив Ubuntu и отключите WSL-компоненты.

### 1.1 Удалите дистрибутив Ubuntu

Откройте PowerShell или Windows Terminal от имени администратора и выполните:

```powershell
wsl -l -v
```

Посмотрите точное имя дистрибутива, например `Ubuntu` или `Ubuntu-24.04`, затем удалите его:

```powershell
wsl --unregister Ubuntu
```

Если у вас другое имя, подставьте его вместо `Ubuntu`.

### 1.2 Отключите компоненты WSL

В том же PowerShell от имени администратора выполните:

```powershell
dism.exe /online /disable-feature /featurename:VirtualMachinePlatform /norestart
dism.exe /online /disable-feature /featurename:Microsoft-Windows-Subsystem-Linux /norestart
```

После этого перезагрузите компьютер.

## 2. Установите WSL2 заново

Самый простой способ — поставить WSL через встроенную команду. Официальная документация Ubuntu рекомендует запускать `wsl --install` из PowerShell от имени администратора. [page:2]

### 2.1 Включите WSL

После перезагрузки откройте PowerShell от имени администратора и выполните:

```powershell
wsl --install
```

Эта команда включает нужные компоненты и ставит Linux-дистрибутив по умолчанию. [page:2]

Если команда не сработала, можно включить компоненты вручную:

```powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

Затем перезагрузите ПК.

### 2.2 Убедитесь, что используется WSL2

После установки выполните:

```powershell
wsl --set-default-version 2
```

Проверьте состояние:

```powershell
wsl -l -v
```

## 3. Установите Ubuntu

Ubuntu можно поставить через Microsoft Store или через команду `wsl --install -d <Distro>`. В документации Ubuntu отдельно упоминается `Ubuntu-24.04` как рекомендуемый LTS-вариант. [page:2]

### Вариант A: через командную строку

Посмотрите доступные дистрибутивы:

```powershell
wsl --list --online
```

Затем установите Ubuntu 24.04:

```powershell
wsl --install -d Ubuntu-24.04
```

### Вариант B: через Microsoft Store

Откройте Microsoft Store, найдите `Ubuntu`, нажмите `Get` и дождитесь установки. После этого запустите Ubuntu из меню Пуск. [page:2]

### 3.1 Первичный запуск Ubuntu

При первом запуске Ubuntu попросит создать пользователя и пароль. Это учетные данные Linux, они могут отличаться от Windows. [page:2]

### 3.2 Обновите пакеты

После входа в Ubuntu выполните:

```bash
sudo apt update
sudo apt full-upgrade -y
```

## 4. Настройте Ubuntu для Cursor
В Ubuntu терминале:

bash
# Обновление системы
sudo apt update && sudo apt full-upgrade -y

# Установка зависимостей для Cursor
sudo apt install -y wget gpg curl libgtk-3-0 libx11-xcb1 libnss3 libasound2 libgbm1 libxss1 libgconf-2-4 xvfb

# Добавление репозитория Microsoft (для VS Code/Cursor)
curl -fsSL https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor | sudo tee /usr/share/keyrings/microsoft.gpg > /dev/null
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/microsoft.gpg] https://packages.microsoft.com/repos/vscode stable main" | sudo tee /etc/apt/sources.list.d/vscode.list

sudo apt update
## 5. Установите Cursor в WSL
## 5.1 Скачайте и установите Cursor
bash
# Установка через официальный скрипт (рекомендуется)
curl https://cursor.com/install -fsS | sh

# Или ручная установка .deb
wget https://www.cursor.com/download/stable/linux-x64 -O cursor.appimage
chmod +x cursor.appimage
./cursor.appimage --version  # проверка
Важно: Cursor в Linux — это AppImage или .deb пакет, работает полностью автономно.

## 5.2 Создайте удобный запуск
bash
sudo tee /usr/local/bin/cursor <<EOF
#!/bin/bash
xvfb-run --auto-servernum cursor.appimage "\$@"
EOF
sudo chmod +x /usr/local/bin/cursor
## 6. Настройка GUI для Cursor в WSL
## 6.1 Установите WSLg (Windows 11) или VcXsrv (Windows 10)
Windows 11 (рекомендуется) — WSLg включено автоматически:

bash
# Просто запустите
cursor --version
Windows 10 — установите VcXsrv:

Скачайте VcXsrv с официального сайта. https://sourceforge.net/projects/vcxsrv/

Запустите XLaunch → Multiple windows → Display 0 → Start no client.

В Ubuntu: export DISPLAY=:0

## 6.2 Создайте Desktop-файл
bash
mkdir -p ~/.local/share/applications
cat > ~/.local/share/applications/cursor.desktop <<EOF
[Desktop Entry]
Version=1.0
Type=Application
Name=Cursor
Comment=AI Code Editor
Exec=cursor %F
Icon=code
Categories=Development;
EOF
## 7. Запуск и проверка
## 7.1 Первый запуск
bash
cursor --no-sandbox --disable-gpu-sandbox
Cursor откроется в отдельном окне с полноценным GUI.

## 7.2 Создайте терминал-алиас
Добавьте в ~/.bashrc:

bash
alias cursor='cursor --no-sandbox'
source ~/.bashrc
## 8. Интеграция с Windows
## 8.1 Доступ к файлам Windows
text
/mnt/c/Users/ВашПользователь/Desktop  # Windows Desktop
/mnt/c/Users/ВашПользователь/Documents # Windows Documents
## 8.2 Открытие проектов
bash
cd ~/projects/myproject
cursor .  # откроет текущую папку в Cursor
## 9. Дополнительная оптимизация
bash
# Увеличьте память WSL (создайте .wslconfig в Windows)
# C:\Users\ВашПользователь\.wslconfig:
# [wsl2]
# memory=8GB
# processors=4

# В Ubuntu — шрифты и темы
sudo apt install -y fonts-firacode fira-code-fonts
Быстрый запуск после настройки
wsl -d Ubuntu-24.04
## Готово!
cursor . — Cursor откроется с GUI!

Преимущества: полная Linux-среда, нет зависимости от Windows-приложений, все обновления через Linux.
